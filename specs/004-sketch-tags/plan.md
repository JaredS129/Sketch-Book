# Implementation Plan: Sketch Tags

**Branch**: `004-sketch-tags` | **Date**: 2026-06-23 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/004-sketch-tags/spec.md`

## Summary

Add a `tags` field (array of lowercase single-word strings) to the sketch data model and surface it throughout the app: a Tags column in the gallery table, a Tags section in the sketch-page metadata panel, and a custom `TagInput` chip-and-autocomplete component in all three sketch forms (create, duplicate, edit). A new `GET /api/tags` Vite-plugin endpoint returns a centralised tag registry (`sketches/tags.json`) that is updated on every sketch save and drives case-insensitive autocomplete without per-keystroke server lookups.

## Technical Context

**Language/Version**: TypeScript 5.7, Node.js 22+ (server middleware), React 19 (browser UI)

**Primary Dependencies**: Existing — Vite 6, React 19, React Router 7, Tailwind v4, TanStack Table, zod, `@radix-ui/react-dialog`. New — `@radix-ui/react-popover` (autocomplete dropdown for TagInput; may already be a transitive peer — check before installing)

**Storage**: Filesystem only — `sketches/<id>/meta.json` (modified), `sketches/tags.json` (new)

**Testing**: Vitest (unit: tag normalisation, registry merge logic, API endpoint tag validation); React Testing Library (component: TagInput chip add/remove, autocomplete open/close)

**Target Platform**: Local developer machine, modern evergreen browser, Vite dev server (`npm run dev` only)

**Project Type**: Web application (frontend SPA) + Node.js dev-server plugin, single npm package

**Performance Goals**: Autocomplete suggestions appear within 1 s of a keystroke (client-side filter on a fetched array — effectively instant on a local machine).

**Constraints**: Tag registry mutations MUST be unavailable in production builds (gated by `apply: 'serve'` on the Vite plugin). `update-sketch-meta` CI fields (`dateUpdated`, `lastUpdatedBy`) MUST remain untouched.

## Implementation Phases

### Phase A — Data model: schema + serialization

**Goal**: `tags` is a valid, optional field on `SketchMeta`. Existing sketches parse without errors. New tags persist to disk correctly.

1. **`scripts/lib/meta.ts`** — add `tags` to `SketchMetaSchema`:
   - Field: `tags: z.array(z.string().min(1)).default([])`
   - In `serializeMeta`: write `tags` only when array is non-empty (same conditional pattern used for `type !== "p5"`).
   - Update `parseMeta` / `safeParseMeta` — no change needed (zod handles it).

2. **`scripts/lib/tags.ts`** — new module (Node.js only, `apply: 'serve'` context):
   - `REGISTRY_PATH`: absolute path to `sketches/tags.json`.
   - `readTagRegistry(): string[]` — reads file (returns `[]` if absent or invalid JSON).
   - `writeTagRegistry(tags: string[]): void` — writes sorted, deduplicated array.
   - `mergeTags(registry: string[], newTags: string[]): string[]` — Set union, lowercased, sorted.
   - `normaliseTags(input: string[]): string[]` — trim + lowercase + deduplicate + reject empties/whitespace/multi-word.

### Phase B — Server: tag registry endpoint + API updates

**Goal**: `GET /api/tags` works. Create, duplicate, and edit endpoints accept and persist tags. Registry updates on every save.

3. **`src/plugins/sketch-api.ts`** — three changes:

   a. **`GET /api/tags`** — new route (before the existing `POST /api/sketches` check):
      ```
      if (method === "GET" && url === "/api/tags")
        → read registry → json(res, 200, registry)
      ```

   b. **`POST /api/sketches` (create)** — read optional `tags` from body:
      - `normaliseTags(body.tags ?? [])` — validate (reject multi-word entries with 400)
      - Pass tags to `writeMeta`
      - `writeTagRegistry(mergeTags(readTagRegistry(), tags))`

   c. **`POST /api/sketches/:sourceId/duplicate`** — read optional `tags`:
      - If `body.tags` present: use `normaliseTags(body.tags)`
      - If absent: inherit from `sourceMeta.tags`
      - Persist to new sketch + merge into registry

   d. **`PATCH /api/sketches/:id` (edit)** — read optional `tags`:
      - `normaliseTags(body.tags ?? [])` — validate
      - Include in `editSketch` call (or write directly after the existing edit logic)
      - Merge into registry

4. **`scripts/lib/edit-sketch-op.ts`** — extend `editSketch` to accept and write `tags`:
   - Add `tags?: string[]` to the options type
   - On `writeMeta`, include `tags` in the written object

### Phase C — TagInput component

**Goal**: A reusable chip-and-autocomplete tag input exists and can be dropped into any form.

5. **Check `@radix-ui/react-popover`** is available (likely a transitive dep already):
   ```bash
   ls node_modules/@radix-ui/react-popover 2>/dev/null || echo "need to install"
   ```
   Install if absent: `npm install @radix-ui/react-popover`

6. **`src/components/TagInput.tsx`** — new component:

   **Props**:
   ```ts
   interface TagInputProps {
     value: string[];               // controlled: current chips
     onChange(tags: string[]): void;
     allTags: string[];             // full registry for autocomplete
     placeholder?: string;
   }
   ```

   **Behaviour**:
   - Renders existing chips as `<span>` pills with an `×` remove button.
   - Inline `<input>` after the last chip.
   - On input change: filter `allTags` case-insensitively against current input → open Popover if ≥1 match.
   - Confirm a tag (add chip) on: Enter, Tab, Space, comma, or clicking a suggestion.
   - Normalise to lowercase before adding; silently ignore duplicates, empty values, and multi-word inputs.
   - Space character: consume the event and add the word typed so far as a chip (prevent space appearing in the field).
   - Popover closes when input is cleared or focus leaves.
   - Already-added tags are excluded from the suggestion list.

   **Styling**: Matches `SketchFormDialog`'s existing input style — `border border-edge bg-surface-2 rounded-md`. Chips use `bg-accent/20 text-accent` (consistent with the project's accent colour token).

### Phase D — Form integration

**Goal**: All three form modes (create, duplicate, edit-full, edit-name-only) include the tags input.

7. **`src/components/SketchFormDialog.tsx`** — four changes:

   a. Extend `SketchFormValues` interface: add `tags: string[]`.

   b. Add `tags` state: `const [tags, setTags] = useState<string[]>(initialValues?.tags ?? [])`.

   c. Add `allTags` prop (fetched by the parent, passed in) or fetch internally on mount via `GET /api/tags`. **Internal fetch on mount is simpler** — no prop threading needed. Use a `useEffect` that calls `fetch('/api/tags').then(r => r.json())` and stores result in local state `allTags`.

   d. Render `<TagInput value={tags} onChange={setTags} allTags={allTags} placeholder="Add tag…" />` in the form, below the Type field (or below Name if Type is hidden). Tags are shown in all form modes.

   e. Include `tags` in the `onSubmit(values)` call.

### Phase E — Display: gallery + sketch page

**Goal**: Tags are visible in the gallery table (with ellipsis overflow) and in the sketch-page metadata panel.

8. **`src/components/SketchTable.tsx`** — add Tags column:
   - New column definition using `columnHelper.accessor('tags', {...})`.
   - Cell renderer: join tags with a comma-space (`tag1, tag2, tag3`).
   - Column cell `className`: `max-w-[160px] truncate` (Tailwind) — forces ellipsis on overflow.
   - Set `title={row.original.tags.join(', ')}` on the cell element for hover tooltip.
   - Position: insert after the Type column, before the Actions column.

9. **`src/components/MetaPanel.tsx`** — add tags row:
   - `META_FIELDS` currently uses `satisfies ReadonlyArray<{ key: keyof SketchMeta; label: string }>`. Since `tags` is now a valid `keyof SketchMeta`, adding it is type-safe.
   - Add `{ key: 'tags', label: 'Tags' }` as the last entry (after `lastUpdatedBy`).
   - In the render: when `key === 'tags'`, render `meta.tags.join(', ')` instead of `meta[key]` directly (array doesn't render as a React node). When the array is empty, render `null` or skip the row.
