# Implementation Plan: Sketch Table Filtering & Search

**Branch**: `Sketch-table-filtering` | **Date**: 2026-06-24 | **Spec**: [spec.md](spec.md)

## Summary

Add four live-filtering controls (name search, type multi-select, tag multi-select, author multi-select) above the gallery table on `HomePage`. All filtering is client-side using a `useSketchFilter` hook that pre-filters `SketchMeta[]` with `useMemo`. A `SketchFilterBar` component renders the controls. No new API endpoints. A shared `MultiSelectDropdown` (Radix Popover + checkboxes) powers the type and author pickers. The existing `TagInput` gains two optional props to support the restricted tag-filter use case.

**Language/Version**: TypeScript 5.7 + React 19 + Vite 6

## Implementation Breakdown

### Step 1 ‚Äî `useSketchFilter` hook (`src/hooks/useSketchFilter.ts`)

New file. Accepts `data: SketchMeta[]`, returns:
- `filter: FilterState` ‚Äî current state `{ name, types, tags, authors }`
- `setFilter` ‚Äî raw setter (passed to `SketchFilterBar`)
- `filteredData: SketchMeta[]` ‚Äî memoised filtered result
- `availableTypes: string[]` ‚Äî de-duped, sorted type values from data
- `availableTags: string[]` ‚Äî de-duped, sorted tags from all sketch tags arrays
- `availableAuthors: string[]` ‚Äî de-duped, sorted createdBy values
- `isFiltering: boolean` ‚Äî any field non-empty
- `clearFilters()` ‚Äî reset to `EMPTY_FILTER`

Filter logic (all AND across filter types):
```
filteredData = data.filter(sketch =>
  (!name || sketch.name.toLowerCase().includes(name.toLowerCase()))
  && (types.length === 0 || types.includes(sketch.type))
  && (tags.length === 0  || tags.every(t => (sketch.tags ?? []).includes(t)))
  && (authors.length === 0 || authors.includes(sketch.createdBy))
)
```

Export `FilterState` and `EMPTY_FILTER` from this file.

---

### Step 2 ‚Äî `MultiSelectDropdown` component (`src/components/MultiSelectDropdown.tsx`)

New file. Uses `@radix-ui/react-popover` (already installed).

Props:
```typescript
{ label: string; options: string[]; selected: string[]; onChange(selected: string[]): void; placeholder?: string }
```

Trigger button text:
- No selection ‚Üí `label` (e.g., "Type")
- N selected ‚Üí `label ¬∑ N` (e.g., "Type ¬∑ 2")

Content: scrollable list of checkbox rows. Each row: `<input type="checkbox">` + label. Clicking a row toggles its value in `selected`.

Styling consistent with existing dark-mode `border-edge bg-surface` tokens.

---

### Step 3 ‚Äî `TagInput` props extension (`src/components/TagInput.tsx`)

Add two optional boolean props:

**`restrictToExisting?: boolean`** (default `false`)  
When `true`, the `addTag` function checks `allTags.includes(tag)` and returns early if false. Keyboard shortcuts (Enter, Tab, comma, space) are also no-ops for non-existent tags.

**`showAllWhenFocused?: boolean`** (default `false`)  
When `true`, suggestions are computed as all unselected `allTags` filtered by `inputValue` substring (including when `inputValue` is empty). Popover opens on focus when there are unselected tags available.

Implement `onFocus` handler on the input that sets `open = true` when `showAllWhenFocused` is true and unselected options remain.

Both props are additive ‚Äî existing call sites pass neither, so behavior is unchanged.

---

### Step 4 ‚Äî `SketchFilterBar` component (`src/components/SketchFilterBar.tsx`)

New file. Stateless ‚Äî all state lives in the hook / parent.

Layout: flex-wrap row with gap. Children in order:
1. **Search input**: `<input type="search">` bound to `filter.name`. `onChange` updates `filter.name` directly.
2. **Type dropdown**: `<MultiSelectDropdown label="Type" options={availableTypes} selected={filter.types} onChange={...}>`
3. **Tag filter**: `<TagInput value={filter.tags} onChange={...} allTags={availableTags} restrictToExisting showAllWhenFocused placeholder="Filter by tag‚Ä¶">`
4. **Author dropdown**: `<MultiSelectDropdown label="Author" options={availableAuthors} selected={filter.authors} onChange={...}>`
5. **Clear button**: `<button disabled={!isFiltering} onClick={onClear}>Clear</button>`. Disabled styling: `opacity-40 cursor-not-allowed`.

Each filter control is labelled (visually or via aria-label) for accessibility. The component accepts the full `SketchFilterBarProps` interface from the contracts doc.

---

### Step 5 ‚Äî `HomePage` integration (`src/pages/HomePage.tsx`)

Modifications:
1. Import and call `useSketchFilter(data)` to get `{ filter, setFilter, filteredData, availableTypes, availableTags, availableAuthors, isFiltering, clearFilters }`.
2. Render `<SketchFilterBar ...>` between the header row and the table/empty-state.
3. Pass `filteredData` to `<SketchTable>` instead of `data`.
4. Update the count line to show:
   - When not filtering: `"{n} sketches ¬∑ click a row to run"`
   - When filtering: `"{filteredData.length} of {data.length} sketches ¬∑ click a row to run"`
5. Empty state: show `<EmptyState>` only when `data.length === 0` (no sketches at all). When filtering produces zero results, show a different "No sketches match your filters" inline message inside the table area (or a minimal empty row message).

## Key Implementation Notes

- `availableTypes` is derived from data at hook call time, not from the `SKETCH_TYPES` constant ‚Äî this ensures only types that actually appear in sketches are shown.
- The tag filter uses AND semantics (all selected tags must be present). This is explicitly documented in the UI (tooltip or helper text is optional but not required by spec).
- The `MultiSelectDropdown` popover should close on outside click (Radix Popover default behavior handles this).
- No debounce is needed for the name search ‚Äî React state updates are synchronous and the filter is a cheap array operation.
- `SketchTable.tsx` receives pre-filtered data and needs no changes.
